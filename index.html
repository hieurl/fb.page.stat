Scenario:
Imagine a game where people posts answer to page's question to get gifts. To avoid too many people having the same correct answer, everyone has to choose a lucky number along with the answer. The one with correct answer and matching random lucky number wins the game.
Eg: Question: 1+1=?
Users' comments:

  • 2. Lucky number: 200
  • 1. Lucky number: 99
  • 2. Lucky number: 123
This tool solves the result filtering problem for that game by generating a random number as lucky number and filter it along with answers to find the winner.
Eg: For previous question, correct answer 2, random lucky number 123 -> 3rd user wins.
Notes:
  • The lucky number must be the last word in the answer for the tool to behave correctly.
  • Answer definition by regular expression supported.
